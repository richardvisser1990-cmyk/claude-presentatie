Het Platform

Drie producten, dezelfde AI

Dezelfde modellen, tools en MCP-verbindingen — maar elk geoptimaliseerd voor een ander type werk.

Claude Chat

Conversatie • Browser • Geen setup

Conversatie in de browser met artifacts: Claude maakt interactieve componenten, grafieken, diagrammen en HTML direct in het gesprek. Geen bestanden op je computer, geen integraties.

Artifacts Projects Web Search
Verschil: Chat maakt artifacts in het gesprek, Cowork maakt echte bestanden op je computer

Cowork

Agentic • Desktop App

Dezelfde engine als Code — file system, code, MCP, sub-agents, skills — maar als desktop app. Maakt echte bestanden, bestuurt je browser, plant taken.

Bestanden Browser MCP Skills
Voor wie: PO's, analisten, managers — iedereen

Claude Code

Agentic • Terminal / VS Code

Volledige agent in je IDE. Leest je codebase, schrijft code, runt tests, maakt PR's. File system, Git, MCP, sub-agents, skills, hooks.

CLAUDE.md Git MCP Skills
Voor wie: Developers — VS Code, terminal, Azure DevOps

Chat is een conversatie-tool met artifacts (interactieve content in het gesprek). Cowork en Code zijn agents — ze draaien dezelfde engine (file system, code, MCP, sub-agents) met een andere interface. Cowork is voor iedereen als desktop app, Code is voor developers in de terminal.

Wat is een skill precies?

        
Een map met minimaal een SKILL.md bestand. Dat is een prompt in markdown die Claude leest voordat het begint te werken. Het kan zo simpel zijn als 10 regels ("gebruik altijd TypeScript strict") of zo uitgebreid als een complete pipeline met scripts, templates, en QA-stappen.

        
Het verschil met een gewone prompt: skills worden automatisch geladen op basis van de taak. Je hoeft niet elke keer je instructies te herhalen.

            mijn-skill/

              optioneel templates/

De SKILL.md

        

          # Pre-commit Quality Gate


          ## Strategie

          Spawn 3 sub-agents parallel voor

          maximale snelheid. Elke agent heeft

          één focus en rapporteert terug.


          ## Agent 1: Security (Explore)

          Gebruik Grep om staged files te scannen:

          - API keys: sk-*, AKIA*, ghp_*

          - Connection strings, .env waarden

          - Hardcoded wachtwoorden

          → BLOCK als gevonden, toon locatie


          ## Agent 2: Quality (Explore)

          Controleer gewijzigde bestanden op:

          - console.log in productie code

          - Uitgecommentarieerde blokken >3 regels

          - Ontbrekende TypeScript return types

          → WARN per bestand


          ## Agent 3: Tests (General Purpose)

          Draai via Bash: npm test

          Check dat nieuwe publieke functies

          een bijhorende test hebben.

          → WARN als test ontbreekt


          ## Combineer resultaten

          Eén BLOCK = commit geweigerd.

          Anders: toon samenvatting en commit.
        

      

    


    

      

        Skills + Sub-agents werken samen: de skill definieert de strategie, de orchestrator leest die en spawnt de juiste sub-agents. Explore agents voor read-only checks, General Purpose voor agents die ook commando's moeten draaien (zoals npm test).

          # PPTX Skill


          ## Kleurpalet keuze

          Kies kleuren passend bij het onderwerp.

          Als je kleuren in een willekeurige andere

          presentatie kan plakken, zijn ze niet

          specifiek genoeg. Nooit default blauw.


          ## Layout regels

          - NOOIT hetzelfde layout herhalen

          - Elke slide heeft een visueel element

          - Wissel: kolommen, kaarten, callouts

          - Geen accent lijnen onder titels

            (= hallmark van AI-generated slides)


          ## QA Pipeline (verplicht)

          1. Genereer slides

          2. Converteer naar afbeeldingen

          3. Start sub-agent voor visuele inspectie

             "Assume there are problems.

             Your job is to find them."

          4. Fix gevonden issues

          5. Herhaal tot alles klopt


          ## Vermijd

          ✗ Text-only slides

          ✗ Centered body text

          ✗ Low contrast elementen

          ✗ Placeholder tekst vergeten

Wanneer agent teams?
